Rapport d'Analyse du Dataset Neurofibromatose (NF1)
TP Deep Learning - Bac4 Génie Logiciel
1. Description Générale du Dataset
1.1 Contexte Médical
Le dataset concerne des cas de neurofibromatose de type 1 (NF1), une maladie génétique caractérisée par :
•	Tumeurs du système nerveux
•	Taches café-au-lait
•	Nodules de Lisch
•	Neurofibromes
1.2 Structure du Dataset
•	296 instances (patients)
•	18 features (caractéristiques cliniques)
•	1 variable cible : Case Type (Familial vs Sporadic)
2. Analyse des Variables
2.1 Variables Démographiques (4 variables)
Variable	Type	Valeurs manquantes	Observations
Age of Mother	Entier	Oui (9.8%)	Âge moyen : 28 ans
Age of Father	Entier	Oui (6.4%)	Âge moyen : 32 ans
Age at First Diagnosis	Entier	Oui (1.7%)	Âge moyen : 11 ans
Tumor Case	Binaire	Non	44.9% avec tumeurs
2.2 Symptômes Cliniques (14 variables binaires)
•	Symptômes cutanés : Café au lait (93.2%), Éphélides (53%)
•	Manifestations ophtalmologiques : Nodules de Lisch (18.6%)
•	Complications neurologiques : Gliome optique (21.3%)
•	Anomalies squelettiques : Dysplasie (10.1%), Scoliose (2.4%)
•	Déficits cognitifs : Trouble d'apprentissage (2.4%)
2.3 Variable Cible
•	Case Type : Familial (1) vs Sporadic (0)
•	Distribution : 55% Familial, 45% Sporadic
3. Problèmes de Qualité des Données
3.1 Valeurs Manquantes
•	Variables parentales : 6-10% de valeurs manquantes
•	Âge au diagnostic : <2% manquants
•	Stratégie recommandée : Imputation par médiane pour les âges
3.2 Incohérences Potentielles
•	Âge au diagnostic > Âge des parents dans certains cas
•	Codage binaire standardisé (0/1) vérifié

5. Préparation pour le Deep Learning
5.1 Pré-traitement Requis
1.	Gestion des valeurs manquantes
o	Imputation médiane pour les variables continues
o	Conservation des variables binaires (pas de valeurs manquantes)
2.	Normalisation
o	Standardisation des variables d'âge
o	Variables binaires déjà normalisées (0/1)
3.	Split des données
o	Train/Validation/Test : 70%/15%/15%
o	Stratification par Case Type
5.2 Features Engineering
•	Création de features : Différence d'âge parental
•	Encodage : One-hot non nécessaire (variables déjà binaires)
•	Sélection : Analyse d'importance des features à prévoir
6. Recommandations pour la Modélisation
6.1 Architectures Recommandées
1.	Réseaux de Neurones Denses
o	Couches : 3-4 couches cachées
o	Unités : 32-128 neurones par couche
o	Fonctions d'activation : ReLU, sigmoid en sortie
2.	Techniques de Régularisation
o	Dropout (20-30%)
o	Batch Normalization
o	Early Stopping
6.2 Métriques d'Évaluation
•	Accuracy : Mesure générale
•	Precision/Recall : Important pour déséquilibre mineur
•	F1-score : Compromis optimal
•	Matrice de confusion : Analyse des erreurs
6.3 Défis Potentiels
1.	Overfitting : Dataset relativement petit
2.	Déséquilibre : Léger (55/45)
3.	Interprétabilité : Importance des features médicales

8. Conclusion
Le dataset présente des caractéristiques intéressantes pour :
1.	Classification binaire familial vs sporadique
2.	Analyse multivariée des symptômes NF1
3.	Détection de patterns cliniques complexes
Points forts : Variables bien définies, codage cohérent, domaine médical pertinent
Limitations : Taille modeste, valeurs manquantes, déséquilibre léger
